SessionStorage

SessionStorage只在本地存储，关闭浏览器后自动删除，存储大小可以达到5M或更大。

兼容性

http://caniuse.com/#search=sessionstorage

API

LocalStorage

LocalStorage只在本地存储，关闭浏览器后不会自动删除，需要手动删除存储数据，存储大小在2.5M到10M之间，各个浏览器大小不同。操作是同步。

兼容性

http://caniuse.com/#search=sessionstorage

API

Cookie

Cookie存储在浏览器端的纯文本，在浏览器和服务端通信过程中来回传递，大小一般是4K，cookie有过期时间，过期时间前关闭浏览器cookie不会失效。

传递条件

  • cookie设置了domain，domain支持通配符配置，如*，当浏览器向domain同源的服务器发起请求，会将浏览器端存储cookie放在request header里面发送给服务端。

属性

  • expires
    cookie过期时间，格式是GMT，可以通过(new Date()).toGMTString()获取GMT时间。已经失效的cookie流量器会自动清理，如果没有设置expires，则默认为session，表示是会话cookie，关闭流量器就会自动清理。

    expires是http 1.0规范，http 1.1使用max-age代替expires

  • domain和path
    domain和path限制cookie能够哪些请求发送会携带该cookie。
    如domain=’*.meituan.com’, path=’/‘，则域名www.meituan.com，api.meituan.com，url为/，/api的请求（包括js/html/css/img，不包括XMLHttpRequest）都会携带该cookie。

    如果是XMLHttpRequest请求，如果非同源请求，则需要考虑跨域资源共享问题。

    同源

    • 协议相同
    • 域名相同
    • 端口号相同

非同源情况下，可以使用cros来实现跨域资源共享。
服务端response header中配置以下请求头

  • Access-Control-Allow-Origin: http://api.bob.com
    支持配置域名访问。

  • Access-Control-Allow-Credentials: true
    是否允许发送cookie，默认不发送cookie。
    还需要浏览器端中配置withCredentials。

  • Access-Control-Expose-Headers: Origin, X-Requested-With, Content-Type, Accept
    配置浏览器端可以从response header中获取哪些属性。

  • httpOnly
    设置httpOnly，只能由服务端设置，发送给客户端，存储在浏览器，浏览器端js无法读取，删除和修改该cookie。

  • secure
    默认情况下，secure选项为空，所以https和http协议都会发送cookie。浏览器端只能在https协议下设置cookie为secure。

  • signed
    给cookie加密，默认是不加密的。

Web SQL

该规范已经在html5规范中废弃了，但是部分浏览器已经实现了该规范，参见w3c文档:https://dev.w3.org/html5/webdatabase/。Web SQL使用sql来操作本地数据库，sql操作是异步的，使用SQLlite。

兼容性

http://caniuse.com/#search=web%20sql

API

  • openDatabase
    参数表：

    1. 数据名名称
    2. 版本号
    3. 描述文件
    4. 数据库大小
    5. 创建回调（可选）

  • transaction
    参数表：

    1. 执行事务回调
    2. 成功回调（可选）
    3. 失败回调（可选）

  • executeSql
    参数表：

    1. 查询sql
    2. 用以替换查询字符串中问号的参数（可选）
    3. 执行成功回调（可选）
    4. 执行失败回调（可选）

IndexedDB

IndexedDB是浏览器端数据库，能够存储大量数据，还能建立索引。
兼容性
http://caniuse.com/#search=indexeddb

特点

  • 键值对存储
    indexedDB是用对象仓库（object store）存储数据，支持JavaScript所有类型数据。数据以key-value键值对的形式保存，key不是重复。

  • 异步
    所有操作都是异步的，数据读写不会阻塞浏览器线程。

  • 支持事务
    支持事务处理，事务失败所有操作回滚。
  • 同域限制
    当前域名只能访问当前域名下数据库，不能跨域名访问数据库。
  • 存储空间大
    一般不少于250，IE是250M，Chrome和Opera是剩余空间的某个百分比，Firefox则没有上限。
  • 支持存储二进制
    不仅可以存储string类型，还支持存储二进制类型。

参考：http://javascript.ruanyifeng.com/bom/indexeddb.html

API

  • open
    参数表：

    1. 数据名
    2. 数据库版本，大于1正整数（可选），默认为1

    onsuccess回调:

  • createObjectStore和objectStoreNames
    创建存放数据的对象仓库，类似关系数据库的table。

    keyPath表示以email作为每条记录的键名。
    autoIncrement表示以自动递增的整数作为键名，从1开始，默认为false。
    一般这两个属性定义一个即可，同时使用则表示以自动递增的整数作为键名，且存储对象必须有email属性。

  • deleteDatabase
    删除添加数据库，关闭浏览器之后数据库不会自动删除，需要手动删除数据库。

  • transaction
    创建事务，处理增删查改。
    参数表:

    1. 对象仓库数组，通常只有一个
    2. 操作类型:readwrite,readonly
