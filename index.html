SessionStorage

SessionStorage只在本地存储，关闭浏览器后自动删除，存储大小可以达到5M或更大。

兼容性

http://caniuse.com/#search=sessionstorage

API

LocalStorage

LocalStorage只在本地存储，关闭浏览器后不会自动删除，需要手动删除存储数据，存储大小在2.5M到10M之间，各个浏览器大小不同。操作是同步。

兼容性

http://caniuse.com/#search=sessionstorage

API

Cookie

Cookie存储在浏览器端的纯文本，在浏览器和服务端通信过程中来回传递，大小一般是4K，cookie有过期时间，过期时间前关闭浏览器cookie不会失效。

传递条件

  • cookie设置了domain，domain支持通配符配置，如*，当浏览器向domain同源的服务器发起请求，会将浏览器端存储cookie放在request header里面发送给服务端。

属性

  • expires
    cookie过期时间，格式是GMT，可以通过(new Date()).toGMTString()获取GMT时间。已经失效的cookie流量器会自动清理，如果没有设置expires，则默认为session，表示是会话cookie，关闭流量器就会自动清理。

    expires是http 1.0规范，http 1.1使用max-age代替expires

  • domain和path
    domain和path限制cookie能够哪些请求发送会携带该cookie。
    如domain=’*.meituan.com’, path=’/‘，则域名www.meituan.com，api.meituan.com，url为/，/api的请求（包括js/html/css/img，不包括XMLHttpRequest）都会携带该cookie。

    如果是XMLHttpRequest请求，如果非同源请求，则需要考虑跨域资源共享问题。

    同源

    • 协议相同
    • 域名相同
    • 端口号相同

非同源情况下，可以使用cros来实现跨域资源共享。
服务端response header中配置以下请求头

  • Access-Control-Allow-Origin: http://api.bob.com
    支持配置域名访问。

  • Access-Control-Allow-Credentials: true
    是否允许发送cookie，默认不发送cookie。
    还需要浏览器端中配置withCredentials。

  • Access-Control-Expose-Headers: Origin, X-Requested-With, Content-Type, Accept
    配置浏览器端可以从response header中获取哪些属性。

  • httpOnly
    设置httpOnly，只能由服务端设置，发送给客户端，存储在浏览器，浏览器端js无法读取，删除和修改该cookie。
    如下图所示：

  • secure
    默认情况下，secure选项为空，所以https和http协议都会发送cookie。浏览器端只能在https协议下设置cookie为secure。

  • signed
    给cookie加密，默认是不加密的。

Web SQL

该规范已经在html5规范中废弃了，但是部分浏览器已经实现了该规范，参见w3c文档:https://dev.w3.org/html5/webdatabase/。Web SQL使用sql来操作本地数据库，sql操作是异步的，使用SQLlite。

兼容性

http://caniuse.com/#search=web%20sql

API

  • openDatabase
    参数表：

    1. 数据名名称
    2. 版本号
    3. 描述文件
    4. 数据库大小
    5. 创建回调（可选）

  • transaction
    参数表：

    1. 执行事务回调
    2. 成功回调（可选）
    3. 失败回调（可选）

  • executeSql
    参数表：

    1. 查询sql
    2. 用以替换查询字符串中问号的参数（可选）
    3. 执行成功回调（可选）
    4. 执行失败回调（可选）

IndexedDB

IndexedDB是浏览器端数据库，能够存储大量数据，还能建立索引。
兼容性
http://caniuse.com/#search=indexeddb

特点

  • 键值对存储
    indexedDB是用对象仓库（object store）存储数据，支持JavaScript所有类型数据。数据以key-value键值对的形式保存，key不是重复。

  • 异步
    所有操作都是异步的，数据读写不会阻塞浏览器线程。

  • 支持事务
    支持事务处理，事务失败所有操作回滚。
  • 同域限制
    当前域名只能访问当前域名下数据库，不能跨域名访问数据库。
  • 存储空间大
    一般不少于250，IE是250M，Chrome和Opera是剩余空间的某个百分比，Firefox则没有上限。
  • 支持存储二进制
    不仅可以存储string类型，还支持存储二进制类型。

参考：http://javascript.ruanyifeng.com/bom/indexeddb.html

API

  • open
    参数表：

    1. 数据名
    2. 数据库版本，大于1正整数（可选），默认为1

    onsuccess回调:

  • createObjectStore和objectStoreNames
    创建存放数据的对象仓库，类似关系数据库的table。

    keyPath表示以email作为每条记录的键名。
    autoIncrement表示以自动递增的整数作为键名，从1开始，默认为false。
    一般这两个属性定义一个即可，同时使用则表示以自动递增的整数作为键名，且存储对象必须有email属性。

  • deleteDatabase
    删除添加数据库，关闭浏览器之后数据库不会自动删除，需要手动删除数据库。

  • transaction
    创建事务，处理增删查改。
    参数表:

    1. 对象仓库数组，通常只有一个
    2. 操作类型:readwrite,readonly

《写给大家看的设计书》读书笔记

发表于 2017-07-03   |   分类于 读书笔记   |  

四大原则

  1. 对比（contrast）
  2. 重复（repetition）
  3. 对齐（alignment）
  4. 亲密性（proximity）

亲密性

将相关的项组织在一起。目的是实现组织性。

注意：

  • 不能因为有空白将元素放在角落或中央
  • 避免页面有太多孤立元素
  • 不要在元素之间留出相同大小空白，除非各组同属一个子集
  • 不同组元素不要建立关系

对齐

任何元素都不能在页面上随意放置，每一项都应当与页面上某个内容存在某种视觉联系。目的是使页面统一而有条理。

初步要求

页面上只使用一种文本对齐：所有文案都左对齐，右对齐或居中对齐

打破对齐

有意识的打破对齐也需要确保符合一定的规则，在打破规则时清楚规则

重复

设计的某些方面需要在整个作品中重复。使用重复目的是统一，并增强视觉效果。

注意：

  • 采用多样性实现统一。
  • 避免太多重复一个元素。

对比

如果两个项不完全相同，就应当使之不同，而且应当是截然不同。使用对比目的是增加页面的效果。

实现

使用字体选择、线宽、颜色、形状、大小、空间等来增加对比。

颜色运用

三原色

红、黄、蓝

三色组

色轮上彼此等距离的三种颜色组成三色组。

分裂互补色

从色轮上选择一种颜色，选择等距离的两种颜色两侧的颜色。

类似色

由色轮上彼此相邻的颜色组成。

暗色和亮色

  • 纯色就是色调（hue）
  • 色调增加黑色构成暗色（shade）
  • 色调增加白色构成亮色（tint）

单色

单色组合：由一种色调以及其相应的多种亮色和暗色组成。

色质

色质：某种颜色的特定名单独、深浅度或色调。

暖色和冷色

暖色：如：红色、黄色，趋进型。
冷色：如：蓝色，后退型。

注意：

  • 不要让冷暖色太多均衡

CMYK与RGB，印刷与Web

CMYK

CMYK代表cyan，magenta，yellow和一个key（通常是黑色）。利用这4中墨色，可以打印出千万种颜色。

RGB

RGB表示red，green和blue。混合red和green可以得到yellow。混合所有颜色可以得到白色，去掉所有颜色得到黑色。

  • red：#f00
  • yellow: #ff0
  • green: #0f0
  • blue: #00f
  • white: #fff
  • black: #000

注意：

  • 需要印刷的项目应当使用CMYK
  • web项目使用RGB

技巧

创建包装

每一件作品中都必须有某个标识性图像或某种标识行风格。

企业名片

除了主色外，如果使用了第二种颜色（辅助色），则用量一定要少。

信笺和信封

使用和企业名片相同的布局，避免使用括号、缩写以及会增加混乱的多余的词。

传单

可以使用有趣的字体和夸张的对比。

步骤：

  • 创建中心点：视觉中心点
  • 使用有对比的子标题
  • 重复
  • 对齐

新闻简报

标题鲜明，每一页内容布局一致。

宣传册

每一页都有明确的对齐线。内容需要逐步吸引读者，需要有对比性。

明信片

  • 与众不同
  • 成“系列”
  • 明确具体
  • 力求简练
  • 尽可能使用颜色

报纸广告

  • 空白
  • 机敏
  • 清楚
  • 简洁
  • 尽可能使用颜色

网站

  • 重复：每一个页面重复某些视觉元素
  • 可读性：行宽要短

字体

字体对比：大小、粗细、颜色、结构、形状、方向。

  • 协调
    只使用一种字体。
  • 冲突
    使用两种或多个类似字体，它们并非完全不同，但是也不完全相同，这个设计就是冲突的。
  • 对比
    使用多种完全不同的字体。

字体类别

oldstyle

基于手写体创建的，有粗/细过渡和一条对角强调线。

modern

有有水平截线，强烈的粗/细过渡和垂直的强调线。

slab serif

也称为clarendon，egyptian，因为是在古埃及文明发展期间盛起，很多都有一个埃及名。粗/细过渡很小甚至没有。
简洁，可读性较好。

sans serif

无截线，无可见粗/细过渡。

script

利用书法笔或书法刷手写的所有字体都是Script类字体。

decorative

使用原则:

  • 不要把同一个类别中的两种字体放在同一个页面上。
  • 永远不要使用全大写

设计过程

  • 从中心点开始

    确定用户最先看什么。

  • 将信息分组

    按逻辑进行分组，确定这些组之间的关系。

  • 创建重复

    找出可以重复的项，可以使用粗字体
    线、装饰符号或某些空间布局。
